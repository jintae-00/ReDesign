ReDesign

Recovering Editable Design Structures from Images via Agentic Decomposition

A Flat Image Is a Dead End for Editing

When the original design file is lost, all that's left is a rendered image. You can no longer move one element, recolor a single shape, fix a typo, or reflow the layout — the pixels don't say which object is which, what attributes produced them, or how they were stacked. Recovering that structure is the gap between "an image of a design" and "a design you can edit."

What ReDesign Does

ReDesign recovers a fully editable design hierarchy from a single image. Starting from the whole image as the root, a vision-language controller grows a layer tree breadth-first (coarse → fine), choosing tool-backed actions at each node and verifying every expansion. The output is an editable JSON hierarchy: text layers with real typography, vector shapes with fill colors, groups with z-order, and isolated elements that survive real edits — delete, recolor, rotate, reflow — without disturbing the rest of the design.

Instead of pixel-space edits that blur and corrupt (like generative image tools), ReDesign exposes discrete, semantic elements that you can manipulate precisely: move a logo 100px right, change a title color, reorder layers in the stack.

We Grow the Design Back, Layer by Layer

ReDesign casts raster-to-editable reconstruction as growing a layer hierarchy. Starting from the whole image as the root, a VLM controller expands the tree breadth-first (coarse → fine). At each node it chooses one tool-backed action that produces child layers; a modular verifier checks every expansion — accepting it, pruning broken or duplicate branches, or retrying with another tool — until every leaf is an atomic, editable element.

Per-Element Reconstruction Quality

Structure You Can Actually Edit

A reconstruction is only useful if its elements survive real edits. We replay six atomic edits — delete, opacity, recolor, rotation, transition, z-order — on each method's output and measure how faithfully the result matches the ground-truth edit.

Qualitative Edits: Before & After

Replaying the same atomic edit on each method's output. ReDesign applies the edit cleanly to the correct element while preserving the rest of the design; baselines move the wrong region, drop elements, or corrupt the layout.

Why Structure Beats Pixel Regeneration

A generative image editor (e.g., Nano Banana 2) edits in pixel space, so precise, controllable design edits — "move this to the bottom-right", "put it behind the desk", "shift up 100 px and rotate 45°" — leak into the rest of the image and lose the original layout. Because ReDesign exposes each element as a discrete, isolated layer with explicit position and rotation, the same instructions execute exactly.
